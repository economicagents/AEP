# AEP — Agent Memory

## Learned User Preferences

- Prefer minimal operational coupling: core flows use direct RPC URLs (user- or env-provided), a self-hosted bundler when needed, and on-chain state; avoid mandatory hosted databases or app-layer reliance on vendor SDKs. Public RPC endpoints and fallbacks belong in environment or config, not secrets managers; optional paid RPC for hosted deployments is fine when wired as plain URLs or keys in env/docs.
- Use forkable references only: eth-infinitism/account-abstraction and resources/awesome-4337.md are for examples/resources/forkable codebases—not as live dependencies or third-party services.
- Prefer lightweight, robust, optimal, secure architectural and codebase design. When design quality is criticized (sizing, spacing, typography, syntax formatting), fix systematically; prefer standardized, congruent visual treatment over excessive variance.
- When filling gaps or making design decisions, consult AGENTS.md, public docs (`docs/ARCHITECTURE.md`, `docs/THREAT-MODEL.md`, `docs/COOKBOOK.md`), and the private ops repo **AEP-Internal** (hosted deploy, runbooks) when you have org access; follow existing patterns. When in doubt, choose the approach most consistent with the project and published docs.
- Continue-implementation and finalize-implementation commands are AEP-specific: use forge (contracts), pnpm (SDK/CLI), not Zig. Validation: `pnpm run test` (runs forge test from contracts/ + pnpm -r test); forge build, forge fmt --check; pnpm run build; align with docs/THREAT-MODEL.md.
- Do not defer, leave TODOs, or note things for later—resolve gaps in the current step.
- Resolve every error and warning; do not suppress, ignore, or cast away type issues unless there is a documented, pre-existing exception.
- Prefer Foundry keystore (cast wallet import); allow PRIVATE_KEY as fallback with warnings. Never store plaintext private keys in .env as the primary path.
- Prefer executing installs, forge, pnpm, and verification commands in the agent environment over instruction-only handoffs when the workspace allows it.
- Chain deploy and hosting procedures: **docs/guides/deployment.md** is the single canonical guide (Sepolia, mainnet gate, post-deploy, operator cheat sheet); do not resurrect parallel deployment markdown paths. **docs/DOCUMENT-MAP.md** indexes public `docs/` and points maintainers to the private **AEP-Internal** repo for EC2/GTM vs engineering routing.
- Repository `.cursorignore` should exclude bulky or generated paths (dependencies, build outputs, vendored Solidity under `contracts/lib/`, lockfiles if desired, local artifacts) so agents focus on first-party code; keep **`.cursor/`** unignored so project rules and commands stay available.

## Learned Workspace Facts

- AEP (Agent Economic Protocol) is the runtime layer for economic agent commerce on ERC-4337, ERC-8004, and x402. Deeper implementation notes may live in the private **AEP-Internal** repo; shipped public truth lives in docs (ARCHITECTURE, THREAT-MODEL, COOKBOOK, BACKLOG for gaps/limitations) and README (implementation summary).
- Monorepo: root package **`@economicagents/workspace`** (private); `contracts/` (Foundry); `packages/*` are **`@economicagents/<folder>`** (sdk, cli, indexer, resolver, graph, mcp, api, monitor, keystore, viem-rpc, smoke-tests, benchmark, web). Runtime config is `~/.aep/config.json` with overrides `AEP_CONFIG_PATH` and `AEP_CHAIN_ID` (84532 Base Sepolia, 8453 Base mainnet). Internal `@economicagents/*` deps use `workspace:*` so pnpm symlinks `packages/*` and dependents resolve built `dist/` after `pnpm -r run build`.
- Economic account stack: smart account, BudgetPolicy, CounterpartyPolicy (allow/block lists, min reputation, getDenyReason), RateLimitPolicy, AEPAccountFactory, PolicyRegistry (Ownable2Step), SDK, CLI, x402 interceptor (reasons include REPUTATION_TOO_LOW), UserOp execute path, frozen kill switch, owner-gated policy changes and UUPS upgrades.
- Intent resolution: Zod intent schema; indexer syncs ERC-8004 into `~/.aep/index` (`aep-index sync`, optional `embed`); hybrid discovery when search index exists else legacy expansion; resolver discover/filter/score/plan with max_total enforcement; MCP `resolve_intent`, REST `POST /resolve`, CLI `aep resolve`, benchmark harness.
- Economic relationships: CreditFacility, ConditionalEscrow (including multi-milestone and factory setupFee), RevenueSplitter, SLAContract and factories; SDK/CLI/MCP coverage for credit, escrow, splitter, and SLA; monetization via factory fees and optional API x402 paywall (treasury env/config)—see docs/COOKBOOK.md and skills/aep-monetization.
- Graph package (sync, credit score, analytics, recommendations; CLI `aep graph *`; resolver optional `accountAddress`+`graphPath` boost; `better-sqlite3` with sql.js fallback; store tracks `dbPath`). MCP and REST: budget/policy/balance, resolve_intent, analytics, credit score, recommendations, fleet summary/accounts/alerts, relationship state; API probe, graphql, analytics pro; `~/.aep/config.json` and optional API keys (`AEP_FLEET_API_KEY`, `AEP_ANALYTICS_PRO_API_KEY`).
- Operations: `packages/monitor` / `aep monitor`; docs/INCIDENT-RESPONSE-PLAYBOOK.md; **docs/guides/deployment.md** (Sepolia + mainnet, local services); `pnpm run validate:testnet`. **`BASE_SEPOLIA_RPC` overrides `rpcUrl` from `~/.aep/config.json`** in smoke tests so a post–mainnet config does not break Sepolia reads. Production managed API: **https://api.economicagents.org** (hosted stack — indexer, graph, API, monitor — per public runbook/deployment docs); on hosted EC2, **systemd** and **cron** must invoke **`node`** with built package entrypoints under **`dist/`** (e.g. monitor CLI), not **`pnpm exec aep`**, so non-interactive services do not fail on PATH or login shells. Hosted runtime config comes from **`/var/lib/aep/config.json`** and **`/var/lib/aep/env`** per deploy/runbook; **AWS Secrets Manager** is not part of that path. Indexer population on the host is proven by **`providers.json`** (and related index files) under the configured index directory; large **`eth_getLogs`** syncs may need a paid or higher-capacity RPC if free tiers time out. For SSH hardening verification on **Amazon Linux 2023**, use **`sudo sshd -T`** for effective OpenSSH settings (**`sshd_config`** is not world-readable). Hosted EC2 access (SSH, SG `/32` drift, **AL2023 minimal** + **Instance Connect**): **AEP-Internal** → `deployment-hosted/docs/ssh-access.md`.
- Base mainnet contract checks: **`pnpm run verify:mainnet-forge`** (forge verify batch) then **`pnpm run verify:mainnet-signoff`** (viem + Etherscan **API v2** `getsourcecode` at `api.etherscan.io/v2/api` with **`chainid=8453`** and `ETHERSCAN_API_KEY`; avoid deprecated per-chain V1 explorer API hosts). Space sequential explorer calls (~3/sec limit); do not wrap sign-off in tight retry loops. Keep **`aepAccountImplementationAddress`** / **`implementationAddress`** in `~/.aep/config.json` when checking the implementation.
- On-chain integration: ERC-4337 core vendored under `contracts/src/vendor/` (eth-infinitism v0.7); OpenZeppelin via forge install; Base Sepolia EntryPoint v0.7 `0x0000000071727De22E5E9d8BAf0edAc6f37da032`; registry and token addresses in deployment docs.
- Security documentation: docs/THREAT-MODEL.md; `audit-report.md` is a post-remediation AI-assisted contract review (not a substitute for a third-party audit)—frame honestly for external audiences; historical Slither and remediation notes live in that report.
- Publishing and repo: publish `@economicagents/*` (CLI global binary **`aep`**) per docs/PUBLISHING.md; `pnpm run publish:packages:dry-run` then `publish:packages` after `npm login`, **`@economicagents`** org access, **npm 2FA**—otherwise **403**; **EOTP** → **`NPM_OTP=<code> pnpm run publish:packages`**. Run `pnpm run verify:npm-metadata` before publishing. `@economicagents/web` is private (site deploy only). Canonical **`https://github.com/economicagents/AEP`** (docs/REPOSITORY.md); open-sourcing: docs/OPEN-SOURCE-RELEASE.md, scripts/secret-scan.sh; CI on `main`/PRs and DCO.
